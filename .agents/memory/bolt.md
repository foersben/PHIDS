## 2024-05-18 - [ECSWorld.query Bottleneck & Safe Iteration]
Learning: Single-component queries in the `ECSWorld` are a significant performance bottleneck due to unnecessary overhead from list comprehensions and computing intersections via `min(sets, key=len)`. However, when iterating over the query results directly or introducing a fast path for single components, it is critical to copy the underlying sets via `list(self._component_index.get(...))` prior to yielding. Failing to do so triggers `RuntimeError: Set changed size during iteration` since systems inside the game loop might add or remove entities or components dynamically while iterating over them.

Action: Always implement specialized fast paths for single-item lookups where set operations are involved. Crucially, strictly enforce `list(...)` copying on internal sets prior to yielding references during any entity-component query operation to ensure safety against concurrent or self-mutating dynamic changes during simulation phases.
