## 2025-03-09 - Extracted Dashboard Presenter God-Class
Learning: The `phids.api.presenters.dashboard.helpers.py` file was functioning as a catch-all "god class," containing pure structural utilities (`_coerce_int`, `validate_cell_coordinates`), substance logic, mycorrhizal payloads, and whole-dashboard assembly (`build_live_dashboard_payload`). Extracting these responsibilities into distinct modules (`shared.py`, `mycorrhizal.py`, `substances.py`, `cell_details.py`, `payloads.py`) strictly segregates UI rendering domains, allowing independent verification.
Action: Next time you notice presentation layers coupling unrelated ECS domains (like substance processing with mycorrhizal network generation) via a generic "helpers" file, aggressively split the logic by domain and verify typing/imports repository-wide, removing backward-compatibility shims.
