Trigger Rules & Gates Configuration Guide

🔄 Rule Lifecycle & Synthesis

The simulation engine evaluates active trigger rules per-cell at each simulation step (tick) following this timeline:

  • Attack Detection: An herbivore swarm of species H_i enters a cell containing plant species P_j. If the swarm population is ≥ the rule's minimum trigger headcount, synthesis begins.
  • Active Synthesis: The plant spends the substance's configured energy cost per tick to produce the chemical. This phase lasts for the specified Synthesis Duration (ticks).
  • Activation Gating: Once synthesis is underway, the engine evaluates the rule's nested Activation Condition. If no condition is configured, the release is unconditional. If a gate exists, the chemical's active effects (repelling, poisoning, or signaling) are blocked until the gate evaluates to True.
  • Dissipation & Aftereffects: When synthesis halts, the chemical continues to provide protection for the duration of the substance's Aftereffect Ticks.

🌿 Logic Gates & Predicates

Gating structures are formed by connecting logical operators (Group nodes) to environmental measurements (Leaf nodes):

  • all_of (AND Gate): Evaluates to True only if all child predicate nodes within this block evaluate to True.
  • any_of (OR Gate): Evaluates to True if at least one child predicate node within this block evaluate to True.
  • herbivore_presence (Leaf): Checks the current cell's local herbivore population. Evaluates to True if the selected herbivore species meets or exceeds the target count.
  • substance_active (Leaf): Checks if the selected substance is currently active (synthesizing or lingering) within the plant's own cell. This allows you to construct cascading defensive pathways (e.g., Substance A must be active to trigger Toxin B).
  • environmental_signal (Leaf): Checks local volatile signaling concentration grids. Evaluates to True if the diffused chemical level at the plant's coordinates exceeds the minimum threshold.

⚡ Applying Config changes

The PHIDS builder is data-oriented. Understanding how changes apply is crucial:

  • Draft State: Adding a rule, deleting nodes, changing values, or configuring gates updates the in-memory DraftState configuration on the server.
  • Live Commit: The active running simulation loop remains isolated from your draft edits. You must click the 🔄 Load Draft button in the top sticky header (or click ▶ Start / ↺ Reset) to compile these rules into Numba-compiled ECS system arrays.
  • Parallel Execution: Multiple rules configured on the same (plant, herbivore) pair fire simultaneously and independently, allowing a single attack to trigger multiple signaling pathways at once.

Passive Morphology Guide

🗡️ Mechanical Damage

Constitutive defenses such as thorns, spines, and trichomes cause direct physical harm to attacking herbivores. This damage is applied instantly per grazing action.

  • Suitable Range: Typically [0.0, 5.0].
  • 0.0: Completely defenseless (soft tissues).
  • 1.0 - 2.0: Moderate defense, deters smaller swarms.
  • > 3.0: Severe physical deterrent (heavy thorn cover).

🧱 Digestibility Modifier

The accumulation of structural polymers like lignin and silica makes the plant tissues harder to digest, directly reducing the caloric/nutritional energy extracted by herbivores.

  • Suitable Range: Must be a multiplier within [0.0, 1.0].
  • 1.0: Fully digestible (100% caloric yield).
  • 0.4 - 0.8: Typical range for armored, woody, or mature tissues.
  • 0.0: Completely indigestible (yields 0 energy to the grazing herbivore).
