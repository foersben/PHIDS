Monte Carlo Batch Runner Guide

🧠 Parallel Monte Carlo Engine

The Batch Runner runs multiple independent simulation trajectories in parallel using a Python ProcessPoolExecutor.

Because individual loops are stochastic (containing random walk vectors, mitotic events, and probabilistic grazing), a single run does not represent the full system space. Running an ensemble reveals the statistical bounds of the ecosystem's state space, including stable attractors, oscillations, and extinction rates.

⚙️ Execution Parameters

  • Ensemble Size (Runs): Controls how many trajectories are simulated (1 to 256). More runs yield smoother aggregate statistical curves and narrower confidence bands.
  • Max Ticks: Limits the depth of each trajectory. Runs terminate immediately if extinction criteria are met beforehand.
  • Draft State Binding: Trajectories are initialized from the active DraftState configuration. Any unsaved edits made in the species tables or Placement Editor are automatically included at launch.

📊 Statistical Outputs & Formats

  • Mean ± Standard Deviation (mean ± σ): Visualized as shaded ribbons, showing the variance of the plant and herbivore populations over time.
  • Extinction Probability: The percentage of runs where plant species population drops to zero.
  • Export Formats: Export the aggregate data via CSV for data processing, LaTeX Table for publication documents, or TikZ (PGFPlots) for vector-graphics rendering in LaTeX layouts.
