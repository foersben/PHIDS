## 2025-06-25 - Focus-Visible Utility Missing on HTMX Delete Buttons
**Learning:** HTMX-injected dynamic elements (like swapped UI buttons, specifically the coordinate-based placement list delete buttons) often lack standard browser focus styles and generic ARIA contexts, significantly impairing keyboard navigability and screen-reader usability for visually impaired scientists/users.
**Action:** Always apply explicit focus-visible classes using Tailwind (e.g., `focus:outline-none focus-visible:ring-2 focus-visible:ring-red-500 rounded`) to ensure keyboard accessibility. Additionally, contextually bind specific coordinate variables to the `aria-label` and `title` tags (e.g., `aria-label="Remove plant at ({{ p.x }}, {{ p.y }})"`) to fully clarify the destructive action's precise target.
## 2024-11-20 - [Icon-Only Buttons A11y] Learning: [Missing focus rings and `aria-label` on icon-only diagnostics buttons] Action: [Always apply `aria-label` along with explicit Tailwind `focus-visible` styling (e.g., `focus:outline-none focus-visible:ring-2 focus-visible:ring-indigo-500`) to all icon-only buttons to ensure they are accessible to both screen readers and keyboard users.]
