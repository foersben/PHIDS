## 2024-05-11 - Adding Context and Confirmation to Icon-Only Destructive Actions
**Learning:** Icon-only destructive actions within lists (like the `✕` delete buttons for plants and swarms in `placement_list.html`) require very specific contextual information for screen readers to differentiate between multiple list items. Adding `aria-label="Remove [item type] at [coordinates]"` prevents the "Delete, Delete, Delete" ambiguity. Furthermore, making them safely interactive requires explicit keyboard focus states (`focus-visible:ring-2`) and immediate confirmation mechanisms (`hx-confirm`) to prevent accidental destructive clicks.
**Action:** When creating new list-based UI components with inline edit or delete actions, immediately pair icon-only buttons with templated context-aware `aria-label`s, `title` attributes, focus styles, and a confirmation step by default.
